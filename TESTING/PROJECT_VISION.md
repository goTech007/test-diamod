## 2. Краткое описание проекта

**Название проекта:** ИИNEUROCHIEF × Фабрика здорового питания  
**Тип проекта:** Пилотный проект по внедрению ИИ-системы для автоматизации и оптимизации производства

### 2.1. Цель проекта

Разработка и внедрение интеллектуальной системы на базе искусственного интеллекта для автоматизации контроля качества, оптимизации производственных процессов и прогнозирования на фабрике здорового питания.

### 2.2. Основные компоненты системы

1. **Система компьютерного зрения (CV)**
   - Контроль персонала (время операций, соблюдение спецодежды)
   - Контроль качества продукции (внешний вид, недовес/перевес)
   - Контроль санитарии (уборка рабочих мест, розливы и рассыпания)

2. **Система прогнозирования и планирования**
   - Планирование выпуска продукции по сменам
   - Прогнозирование спроса
   - Планирование загрузки оборудования
   - Планирование смен персонала

3. **Аналитическая система**
   - Мониторинг загрузки производственных линий
   - Мониторинг загрузки персонала
   - Отслеживание объемов упакованной и маркированной продукции
   - Анализ простоев оборудования
   - Выявление проблем перепроизводства (особенно обечаек)

### 2.3. Приоритетные зоны для пилота

- **Цех упаковки** — основная зона пилота
- **Участок маркировки продукта и передача на склад** — приоритет #1
- **Участок расфасовки полуфабрикатов** — приоритет #2

### 2.4. Ключевые задачи пилота

1. Получение метрик времени простоя оборудования
2. Планирование работы по матрице от плана производства с установкой времени начала упаковки к заданному времени отгрузки на склад
3. Контроль времени операций (например, сколько стоит лоток в ящике после одевания обечайки или этикетки)
4. Контроль отсутствия маски при расфасовке полуфабрикатов
5. Снижение списаний обечаек
6. Оптимизация планирования персонала и оборудования

## 4. Принципы разработки

- **Модульность** — система должна состоять из независимых, переиспользуемых компонентов
- **Масштабируемость** — архитектура должна поддерживать рост нагрузки и функциональности
- **Безопасность** — соблюдение лучших практик безопасности на всех уровнях
- **Надежность** — обеспечение отказоустойчивости и восстановления после сбоев
- **Производительность** — оптимизация для обеспечения быстрой работы системы
- **Поддерживаемость** — код и архитектура должны быть понятными и легко поддерживаемыми
