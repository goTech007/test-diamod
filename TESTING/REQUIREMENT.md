2. Задача

На основе данных брифа самостоятельно сформировать полноценное видение проекта, включающее:

2.1. Архитектуру проекта

Необходимо представить архитектуру в текстовом виде (диаграммы можно описать словами), раскрывая:
 • Общую концепцию и логику работы системы
 • Архитектурный стиль (микросервисы, монолит, гибрид — обосновать выбор)
 • Стек технологий (backend, frontend, БД, инфраструктура, авторизация, платежи, интеграции и др.)
 • Модули системы и их взаимодействие
 • Хранилища данных и ключевые сущности
 • Типы интеграций с внешними системами
 • Требования к инфраструктуре, CI/CD и безопасности
 • Возможности масштабирования и отказоустойчивости

⸻

2.2. План разработки

Разработать подробный roadmap проекта, состоящий из:
 • Этапов реализации (аналитика, проектирование, разработка, тестирование, релизы)
 • Декомпозиции по функциональным блокам
 • Оценки сроков по каждому этапу (в неделях или спринтах)
 • Оценки рисков и зон неопределённости
 • Предложений по MVP и дальнейшему развитию продукта

⸻

2.3. Необходимые ресурсы

Указать:
 • Кто требуется для реализации проекта (разработчики, дизайнер, аналитик, QA, DevOps и др.)
 • Загрузка специалистов
 • Оценка человеко-часов / человеко-недель
 • Необходимые сервисы/подписки/инфраструктура

⸻

2.4. Данные, которые необходимо получить от заказчика

Сформировать перечень данных, без которых невозможно начать проектирование и разработку, включая:
 • Детализацию бизнес-процессов
 • Правила работы интеграций
 • Пример REST API или ТЗ от внешних сервисов
 • UX-материалы, бренд-гайд
 • Полный список ролей пользователей и их прав
 • Требования к аналитике, логированию, админ-панели
 • Финальные требования к отчётности
 • Юридические требования (хранение данных, персональные данные)

⸻

2.5. Формат результата

Один документ в любом удобном формате, содержащий:
 1. Архитектуру проекта
 2. План разработки
 3. Перечень необходимых ресурсов
 4. Перечень данных/доступов от заказчика
 5. При необходимости — схемы (можно в виде описаний)

Документ должен быть:
 • Чётко структурирован
 • Логичен
 • Сформирован полностью на основе вашего профессионального опыта и анализа брифа
 • Без привязки к реальным командам, прошлым проектам, сторонним подсказкам — вы принимаете все решения самостоятельно

⸻

3. Требования
 • Время выполнения: 4–6 часов
 • Решение должно выглядеть так, как будто вы готовите внутреннее ТЗ для старта проекта перед командой
 • Оцениваться будет: глубина проработки, логика, структурированность и реалистичность решений
